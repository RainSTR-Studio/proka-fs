# Contributing to ProkaFS

First of all, thanks for your contrubutions!

In order to make this project better, we welcome any kind of contributions, such as:

 - **Reporting bugs**: If you find any bugs, please report them in the [issue tracker](https://github.com/RainSTR-Studio/proka-fs/issues).
 - **Suggesting features**: If you have any ideas to improve this project, please let us know in the [issue tracker](https://github.com/RainSTR-Studio/proka-fs/issues).
 - **Submitting pull requests**: If you want to contribute code to this project, please submit a pull request. We will review your code and merge it if it is good.

 ## Code style

 We uses [Rust's code style](https://doc.rust-lang.org/1.0.0/style/index.html) for this project.

 **Simultaneously**, ensure your definition of `struct`, `enum`, `function`, `module`, `trait` is clear and consistent with the code style.

 ## How-to-do

 1. **Fork** this repository to your own account;
 2. **Clone** your forked repository to your local machine;
 3. **Create** a new branch for your changes (e.g. `fix-bug-123`);
 4. **Add** your name to [these positions](README.md#leave-your-name-here);
 5. **Commit** your changes to the new branch;
 6. **Push** your changes to your forked repository;
 7. **Create** a pull request to this repository;
 8. **Wait** for the review and merge.

 If your pull request is not merged immediately, please don't be discouraged. We will review your code and merge it as soon as possible.

 If this project becoming popular, you can say it proudly:

 ***Oh bro, I just join the development of a project, which is very popular!!***

Anyway, thanks for your contributions! We encourage you to do contributions more!!!
